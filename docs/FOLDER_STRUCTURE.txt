================================================================================
COMPONENT BREAKDOWN
================================================================================

MAIN PROCESS (Node.js)
├── fileSystem/
│   ├── operations.js      → fs.promises (copy, move, delete, rename, mkdir)
│   ├── watcher.js         → chokidar (watch file changes)
│   ├── search.js          → fast-glob (search files)
│   └── permissions.js     → fs.access (check permissions)
│
├── archive/
│   ├── zipHandler.js      → adm-zip (zip operations)
│   ├── rarHandler.js      → node-7z (rar/7z support)
│   └── index.js           → Unified archive API
│
├── preview/
│   ├── imagePreview.js    → sharp (thumbnail generation)
│   ├── videoPreview.js    → ffmpeg (video thumbnail)
│   ├── audioPreview.js    → music-metadata (audio info)
│   ├── documentPreview.js → fs.readFile (text preview)
│   └── pdfPreview.js      → pdf-parse (PDF preview)
│
├── ipc/
│   ├── fileHandlers.js    → Handle file operations
│   ├── searchHandlers.js  → Handle search requests
│   ├── previewHandlers.js → Handle preview requests
│   ├── settingsHandlers.js→ Handle settings read/write
│   └── index.js           → Register all handlers
│
├── storage.js             → Read/write JSON files (settings, bookmarks)
├── windowManager.js       → Create/manage windows
└── menuManager.js         → Create app menu

RENDERER PROCESS (Browser)
├── core/
│   ├── app.js             → Initialize app, load settings
│   ├── ipc.js             → Wrapper for window.vortexAPI
│   ├── storage.js         → LocalStorage wrapper
│   └── events.js          → Event bus (EventEmitter pattern)
│
├── ui/
│   ├── header.js          → Render header, handle nav buttons
│   ├── sidebar.js         → Render sidebar, handle clicks
│   ├── tabs.js            → Tab creation, switching, closing

│   └── dialogs.js         → Modal dialogs (progress, conflict, etc.)
│
├── features/

│   ├── dragDrop.js        → Drag files, drop zones
│   ├── copyPaste.js       → Copy/cut/paste logic
│   ├── undoRedo.js        → Undo/redo stack
│   ├── bookmarks.js       → Add/remove/load bookmarks
│   ├── search.js          → Search logic, filters
│   ├── preview.js         → Show file preview
│   ├── shortcuts.js       → Keyboard shortcut handler
│   └── themes.js          → Switch themes, apply accent color
│

================================================================================
DATA FLOW
================================================================================

USER ACTION → RENDERER → IPC → MAIN → FILE SYSTEM
                ↓                ↓
            UPDATE UI ← IPC ← RESPONSE

Example: Navigate to folder
1. User clicks folder in file list
2. fileList.js → handleFileDoubleClick()
3. navigation.js → navigateTo(path)
4. ipc.js → invoke('fs:readDir', path)
5. main/ipc/fileHandlers.js → handle request
6. main/fileSystem/operations.js → readDirectory(path)
7. Return files array
8. fileList.js → render(files)
9. addressBar.js → render(path)
10. footer.js → updateDriveInfo()

================================================================================
STATE MANAGEMENT
================================================================================

Global State (in main.js):
  - currentPath: string
  - currentView: 'grid' | 'list' | 'details'
  - selectedFiles: Set<string>
  - clipboard: { action: 'copy' | 'cut', files: string[] }
  - history: { back: string[], forward: string[] }
  - tabs: Tab[]
  - activeTabId: string
  - settings: Settings
  - bookmarks: Bookmark[]

Tab State:
  - id: string
  - path: string
  - history: string[]
  - historyIndex: number
  - scrollPosition: number
  - selectedFiles: Set<string>
