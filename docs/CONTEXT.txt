================================================================================
CODING STANDARDS (ENFORCED FROM NOW ON)
================================================================================

COMPONENT STRUCTURE RULE:
Every new feature must be broken into small focused components:

  js/features/featureName/
  ├── index.js          ← entry point, wires everything, no logic
  ├── components/
  │   ├── componentA.js
  │   └── componentB.js
  ├── utils/
  │   └── helpers.js
  └── styles/           (or in css/ at root level)

RULES:
  - 1 file = 1 responsibility
  - No file over ~150 lines if it can be split
  - index.js only wires/exports, no logic
  - CSS in its own file per feature
  - Utils separated from UI logic
  - Constants/config in their own file if reused
  - SVG icons ONLY — no emojis anywhere
  - All commits under 20 words

================================================================================
IPC CHANNELS (main process)
================================================================================

fs:readDir       → { success, files: [{name, isDirectory, size, modified, ...}] }
fs:stat          → { success, isDirectory, size, modified }
fs:getDrives     → { success, drives: [{letter, path, size, freeSpace, isPortable, type}] }
fs:getSpecialPath → (name: 'desktop'|'documents'|'downloads'|'pictures'|'music'|'videos') → path string
fs:mkdir         → { success }
fs:createFile    → { success }
fs:rename        → { success }
fs:delete        → { success }
fs:copy          → { success }
fs:move          → { success }
fs:exists        → boolean
fs:search        → { success, files }
window:minimize  → void
window:maximize  → void
window:close     → void
